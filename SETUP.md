# ClientSpring GHL Onboarding Tool — Setup Guide

## What This Does

Watches your Google Drive for new client folders generated by ClientSpring AI.
When a new folder appears, Steve reviews it in the admin panel and clicks **▶ Run Onboarding** to:

1. Create a new GHL sub-account
2. Apply the Master Snapshot (`Client Spring MASTER V2`)
3. Parse Sales Page Copy, Thank You Emails, and Upsell docs from the Drive folders
4. Populate all funnel custom values
5. Create products (main offer + upsells)
6. Create post-purchase thank you email templates

---

## Prerequisites

### 1. Google Service Account Key

The tool uses a Google Service Account to read Drive folders.

- Service account email: `clientspring-nextjs@clientspring.iam.gserviceaccount.com`
- **You need the JSON key file** for this account from Google Cloud Console → IAM → Service Accounts
- Place it in the project root as `google-service-account.json`
- Or set `GOOGLE_SERVICE_ACCOUNT_KEY=/path/to/key.json` in `.env`

**Important:** The service account needs **Viewer** access to all client Drive folders.
If folders are created by ClientSpring AI, make sure the service account is a member of the parent folder.

### 2. Watch Folder ID

Set `WATCH_FOLDER_ID` in `.env` to the parent Google Drive folder where ClientSpring AI drops new client folders.

```
WATCH_FOLDER_ID=1xxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

To find the folder ID: open the folder in Drive, the ID is the last part of the URL:
`https://drive.google.com/drive/folders/**FOLDER_ID_HERE**`

## Running the Server

```bash
npm install
npm start
```

Server starts at `http://localhost:3000`

- **Manual form:** `http://localhost:3000/`
- **Admin panel:** `http://localhost:3000/admin`

---

## GHL Webhook (Automatic from Form Submission)

To automatically queue clients from your GHL intake form:

1. Deploy this server publicly (Railway, ngrok, etc.)
2. In GHL → your intake form → Settings → Notifications → Webhook
3. Set the webhook URL to: `https://YOUR_DOMAIN/webhook/ghl-form`

When a client submits the form, they'll appear in `/admin` for your review.

## Testing Drive Parsing

Before running a full onboard, test that the Drive folder parses correctly:

```bash
# Test offer folder (Sales Page Copy + Thank You Email)
node test-parse.js "https://drive.google.com/drive/folders/YOUR_FOLDER_ID" offer

# Test backend folder (Upsell docs)
node test-parse.js "https://drive.google.com/drive/folders/YOUR_FOLDER_ID" backend
```

---

## Folder Structure Expected

**Offer & Product folder** (from ClientSpring AI):
- `Sales Page Copy` or similar → extracts headline, price, product name
- `Thank You Email` → extracts email subject, body, sets download URL = doc URL

**Backend Engine folder** (optional):
- `Upsell 1 Sales Page` → upsell 1 details
- `Upsell 1 Thank You` → upsell 1 email
- `Upsell 2 Sales Page` → upsell 2 details
- `Upsell 2 Thank You` → upsell 2 email
